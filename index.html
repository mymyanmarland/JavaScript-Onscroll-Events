JavaScript Onscroll Events

ပြည့်စုံသော လမ်းညွှန်ချက် (မြန်မာဘာသာ)

ခလုတ်နှိပ်စရာမလို၊ Mouse Scroll ဆွဲရုံဖြင့် အလုပ်လုပ်သော နည်းပညာများ


Created for Web Developers

မိတ်ဆက်

ယနေ့ခေတ် Web Design များတွင် Scrolling သည် အလွန်အရေးပါသော အခန်းကဏ္ဍမှ ပါဝင်လာပါသည်။

User များသည် Page ကို အပေါ်အောက် ဆွဲချ/ဆွဲတင် လုပ်သည့်အခါ လှပသော Animation များ၊ အသစ်ပေါ်လာသော Data များကို တွေ့မြင်ရလေ့ရှိပါသည်။

DOM Events ဆိုတာ ဘာလဲ?

  • User က Web Page ပေါ်မှာ ပြုလုပ်လိုက်တဲ့ လှုပ်ရှားမှု (Actions) တွေကို ခေါ်ပါတယ်။
  • ဥပမာ - Click နှိပ်ခြင်း၊ Keyboard ရိုက်ခြင်း၊ Mouse ရွှေ့ခြင်း စသည်တို့ဖြစ်ပါသည်။
  • ထိုအထဲမှ အသုံးအများဆုံးတစ်ခုမှာ Scroll Event ဖြစ်ပါသည်။

Onscroll Event ဆိုတာ?

User က Mouse Wheel ကိုလှိမ့်၍ဖြစ်စေ၊ Phone Screen ကို ပွတ်ဆွဲ၍ဖြစ်စေ Web Page (သို့) Element တစ်ခုကို အပေါ်/အောက် ရွှေ့လိုက်သောအခါ အလုပ်လုပ်မည့် Event ဖြစ်ပါသည်။

ဘာကြောင့် Scroll Event ကို သုံးတာလဲ?

UI/UX ပိုကောင်းစေရန်

Data ကို လိုမှ ခေါ်ရန် (Lazy Load)

အခြေခံ ရေးသားနည်းများ

Scroll Event ကို JavaScript တွင် နည်းလမ်း (၃) မျိုးဖြင့် ရေးသားနိုင်ပါသည်။

  1. HTML Inline Attribute အဖြစ်ရေးခြင်း
  2. DOM Element Property အနေဖြင့်ရေးခြင်း
  3. addEventListener အသုံးပြုခြင်း (အကောင်းဆုံးနည်းလမ်း)

၁. HTML Inline Attribute

HTML Tag ထဲတွင် တိုက်ရိုက် ရေးသားသောနည်း ဖြစ်ပါသည်။

<body onscroll="myScrollFunction()">
  <!-- Content Here -->
</body>

* ယနေ့ခေတ်တွင် ဤနည်းကို အသုံးမပြုတော့ပါ။ (Not Recommended)

၂. DOM Property နည်းလမ်း

JavaScript ဘက်မှနေ၍ DOM ကို လှမ်းခေါ်ပြီး ရေးသောနည်းဖြစ်ပါသည်။

window.onscroll = function() {
  console.log("Scrolling...");
};

အားနည်းချက် - Event တစ်ခုတည်းသာ ထည့်သွင်းနိုင်ပါသည်။ (Overwritten ဖြစ်လွယ်သည်)

၃. addEventListener (အကောင်းဆုံး)

Modern Web Development တွင် အသုံးပြုရန် အသင့်တော်ဆုံး နည်းလမ်းဖြစ်ပါသည်။

window.addEventListener('scroll', function() {
  console.log("User is scrolling!");
});

addEventListener ရဲ့ အားသာချက်များ

  • Function များစွာကို Event တစ်ခုထဲတွင် ထည့်ရေးနိုင်ခြင်း။
  • removeEventListener ဖြင့် ပြန်လည်ဖြုတ်ပစ်နိုင်ခြင်း။
  • Performance ကို ပိုမိုထိန်းချုပ်နိုင်ခြင်း (ဥပမာ - { passive: true })။

Window Object နဲ့ Scroll

Web Page တစ်ခုလုံးကို Scroll ဆွဲသည်ကို သိချင်လျှင် window object ကို အသုံးပြုရပါသည်။

Browser window တစ်ခုလုံး၏ ရွေ့လျားမှုကို တိုင်းတာခြင်းဖြစ်ပါသည်။

window.scrollY ဆိုတာ ဘာလဲ?

Document ၏ အပေါ်ဘက် (Top) မှနေ၍ အောက်သို့ Pixel မည်မျှ ရွှေ့သွားပြီလဲ ဆိုသည်ကို ဖော်ပြပေးသော Property ဖြစ်ပါသည်။

let topOffset = window.scrollY;
console.log("Scrolled from top: " + topOffset + "px");

window.scrollX

အကယ်၍ Website သည် ဘေးတိုက် (Horizontal) Scroll ပါဝင်ပါက၊ ဘယ်ဘက်မှ ညာဘက်သို့ Pixel မည်မျှရွှေ့သွားကြောင်း တိုင်းတာပေးပါသည်။

let leftOffset = window.scrollX;
console.log("Scrolled from left: " + leftOffset + "px");

scrollY vs pageYOffset

window.pageYOffset သည် window.scrollY ၏ နာမည်ဟောင်း (Alias) သာဖြစ်ပါသည်။

Browser အဟောင်းများအတွက် pageYOffset ကို သုံးခဲ့ကြသော်လည်း ယခုအခါ scrollY ကိုသာ အများဆုံး သုံးကြပါသည်။

Window Scroll ဥပမာ

Scroll ဆွဲတိုင်း မည်မျှရောက်နေပြီကို စစ်ဆေးခြင်း။

window.addEventListener('scroll', () => {
  if (window.scrollY > 500) {
    console.log("Passed 500 pixels!");
  }
});

Element-level Scrolling

Window တစ်ခုလုံးမဟုတ်ဘဲ၊ Box (သို့) Div အသေးလေးတစ်ခုအတွင်းတွင် Scroll ဆွဲခြင်းကိုလည်း ဖမ်းယူနိုင်ပါသည်။

၎င်းအတွက် ထို Element တွင် overflow: auto သို့မဟုတ် scroll ရှိရန်လိုအပ်ပါသည်။

element.scrollTop

ရွေးချယ်ထားသော Element ၏ အတွင်းဘက်တွင် အပေါ်မှနေ၍ အောက်သို့ Pixel မည်မျှ Scroll လုပ်ထားသလဲကို ပြသပါသည်။

let myBox = document.getElementById("box");
myBox.addEventListener("scroll", function() {
  console.log(this.scrollTop);
});

element.scrollLeft

Element ၏ အတွင်းဘက် ဘယ်မှညာသို့ Scroll ပမာဏကို တိုင်းတာခြင်းဖြစ်ပါသည်။ ဇယား (Table) များတွင် အသုံးများပါသည်။

let myTable = document.getElementById("tableContainer");
console.log(myTable.scrollLeft);

element.scrollHeight

မျက်စိဖြင့် မြင်ရသောအပိုင်းရော၊ Scroll ဆွဲမှ မြင်ရမည့် ဖုံးကွယ်နေသောအပိုင်းပါ ပေါင်းထားသည့် Content တစ်ခုလုံး၏ အမြင့် (Total Height) ဖြစ်ပါသည်။

element.clientHeight

Element အတွင်း မျက်စိဖြင့် လက်ရှိမြင်နေရသော Screen Area ၏ အမြင့် (Visible Height) သီးသန့်ကိုသာ ဆိုလိုပါသည်။ Border များ မပါဝင်ပါ။

ScrollHeight vs ClientHeight

Property အဓိပ္ပါယ်
scrollHeight စာကြောင်းအားလုံး (မမြင်ရတာတွေပါ) အပေါင်းအမြင့်
clientHeight လက်ရှိ ဖန်သားပြင်ပေါ် မြင်နေရတဲ့ အမြင့်သီးသန့်
scrollTop အပေါ်ကို ရောက်သွားတဲ့ (ဖုံးသွားတဲ့) ပမာဏ

လက်တွေ့အသုံးပြုမှုများ (Use Cases)

Scroll Events ကို အသုံးပြု၍ ဖန်တီးလေ့ရှိသော နာမည်ကြီး Features (၄) ခုကို လေ့လာကြည့်ကြပါမည်။

Use Case 1: Sticky Navbar

Website ကို အောက်သို့ ဆွဲချလိုက်သောအခါ Navigation Menu Bar သည် ပျောက်မသွားဘဲ အပေါ်ထိပ်တွင် ကပ်ပါလာခြင်း (Sticky) ဖြစ်ပါသည်။

Sticky Nav အလုပ်လုပ်ပုံ

  1. Window ကို Scroll ဆွဲလိုက်တိုင်း window.scrollY ကို စစ်ဆေးပါမည်။
  2. သတ်မှတ်ထားသော အမြင့် (ဥပမာ 100px) ထက် ကျော်လွန်သွားပါက...
  3. Navbar ထဲသို့ sticky ဆိုသော CSS class တစ်ခု ထည့်ပေးပါမည်။

Sticky Nav (Code Example)

const nav = document.querySelector('nav');

window.addEventListener('scroll', () => {
  if (window.scrollY > 100) {
    nav.classList.add('sticky');
  } else {
    nav.classList.remove('sticky');
  }
});

Use Case 2: Infinite Scrolling

Facebook (သို့) TikTok ကဲ့သို့ အောက်ဆုံးသို့ ရောက်သွားတိုင်း နောက်ထပ် Data အသစ်များ အလိုအလျောက် ပေါ်လာသော စနစ်ဖြစ်ပါသည်။

Infinite Scroll သဘောတရား

  • User သည် Web Page ၏ အောက်ခြေဆုံး (Bottom) သို့ ရောက်မရောက် အမြဲစစ်ဆေးနေရပါမည်။
  • အောက်ဆုံးရောက်သွားပါက Server ဆီမှ (API ဖြင့်) နောက်ထပ် Data များကို လှမ်းတောင်းပါမည်။

အောက်ဆုံးရောက်မရောက် စစ်ဆေးခြင်း

ယင်းအတွက် အရေးကြီးသော Property (၃) ခုလိုပါသည်။

  • window.innerHeight (လက်ရှိမြင်ရတဲ့အမြင့်)
  • window.scrollY (Scroll ဆွဲချလိုက်တဲ့ပမာဏ)
  • document.body.offsetHeight (Page တစ်ခုလုံးရဲ့ အမြင့်)

တွက်ချက်မှု ပုံသေနည်း (Math Logic)

( scrollY + innerHeight ) >= offsetHeight

အပေါ်ပါ နှစ်ခုပေါင်းလဒ်သည် Document အမြင့်နှင့် ညီသွားလျှင် (သို့) ကြီးသွားလျှင် အောက်ဆုံးသို့ ရောက်ပြီ ဟု သတ်မှတ်နိုင်ပါသည်။

Infinite Scroll (Code Example)

window.addEventListener('scroll', () => {
  let scrollable = document.documentElement.scrollHeight - window.innerHeight;
  let scrolled = window.scrollY;

  if (Math.ceil(scrolled) === scrollable) {
    alert("You reached the bottom! Loading more...");
  }
});

Use Case 3: Scroll Animations

Web Page ကို အောက်သို့ ဆွဲချလိုက်စဉ် ပုံများ၊ စာများက Fade In, Slide In စသည့် Animation များဖြင့် လှပစွာ ထွက်လာခြင်းကို ခေါ်ပါသည်။

Element ကို မြင်ရ/မမြင်ရ စစ်ဆေးခြင်း

Scroll ဆွဲလိုက်တိုင်း Element သည် Browser Viewport (လက်ရှိမြင်ကွင်း) ထဲသို့ ဝင်လာခြင်း ရှိ/မရှိ စစ်ဆေးရန် လိုအပ်ပါသည်။

ယင်းအတွက် getBoundingClientRect() ကို အသုံးပြုလေ့ရှိပါသည်။

getBoundingClientRect() ဆိုတာ?

  • Element ၏ အရွယ်အစားနှင့် Browser Window အပေါ်မူတည်၍ တည်နေရာ (Position) ကို ပြန်ပေးသော Function ဖြစ်ပါသည်။
  • rect.top သည် Screen အပေါ်ဘက်နှင့် Element အကြား အကွာအဝေးကို ပြပါသည်။

Animation Class ထည့်သွင်းခြင်း

rect.top သည် Window ၏ အမြင့် (innerHeight) အောက် ငယ်သွားလျှင် (ဆိုလိုသည်မှာ Viewport ထဲ ဝင်လာလျှင်) ထို Element ကို CSS 'show' class ထည့်ပေးရပါမည်။

Scroll Animation (Code Example)

let boxes = document.querySelectorAll('.box');

window.addEventListener('scroll', () => {
  let triggerBottom = window.innerHeight * 0.8;

  boxes.forEach(box => {
    let boxTop = box.getBoundingClientRect().top;
    if(boxTop < triggerBottom) {
      box.classList.add('show');
    }
  });
});

Use Case 4: Scroll Progress Bar

စာဖတ်နေစဉ် အပေါ်ဆုံး နားတွင် Line လေးတစ်ခု ပြေးနေပြီး ဆောင်းပါး ဘယ်လောက်ဖတ်ပြီးသွားပြီလဲ ပြပေးသော Feature ဖြစ်ပါသည်။ (Medium ကဲ့သို့ Blog များတွင် တွေ့ရသည်)

Progress Bar သဘောတရား

  • Scroll ဆွဲနိုင်သည့် စုစုပေါင်း အကွာအဝေးကို ရှာပါ။
  • လက်ရှိ Scroll ဆွဲပြီးသား အကွာအဝေးကို ရှာပါ။
  • ထိုနှစ်ခုကို စားပြီး ၁၀၀ ဖြင့် မြှောက်ကာ ရာခိုင်နှုန်း (Percentage) ရှာပါ။

ရာခိုင်နှုန်း တွက်ချက်ခြင်း

let winScroll = document.body.scrollTop || document.documentElement.scrollTop;
let height = document.documentElement.scrollHeight - document.documentElement.clientHeight;
let scrolled = (winScroll / height) * 100;

Width ပြောင်းလဲပေးခြင်း

တွက်၍ရလာသော ရာခိုင်နှုန်းကို HTML တွင် ကြိုဆောက်ထားသော Progress Bar <div> ၏ CSS Width အဖြစ် ထည့်သွင်းပေးရပါမည်။

Progress Bar (Code Example)

window.onscroll = function() {
  let winScroll = document.documentElement.scrollTop;
  let height = document.documentElement.scrollHeight - document.documentElement.clientHeight;
  let scrolled = (winScroll / height) * 100;
  document.getElementById("myBar").style.width = scrolled + "%";
};

Performance ပြဿနာများ

Scroll Event ကို သတိမထားဘဲ အသုံးပြုလျှင် Website လေးလံသွားခြင်း (Lagging/Janky UI) များ ဖြစ်ပေါ်နိုင်ပါသည်။

Scroll က ဘာလို့ လေးတာလဲ?

User က Mouse Wheel ကို တစ်ခါလှိမ့်လိုက်တိုင်း Scroll Event သည် စက္ကန့်ပိုင်းအတွင်း အကြိမ်ပေါင်းများစွာ (Dozens of times) အလုပ်လုပ်သွားသောကြောင့် ဖြစ်ပါသည်။

Function များ အလုပ်များလွန်းသွားခြင်း ဖြစ်သည်။

ဖြေရှင်းနည်း ၁: Debouncing

Event များ ဆက်တိုက်ဝင်လာတိုင်း အလုပ်မလုပ်သေးဘဲ၊ User Scroll ဆွဲတာ ရပ်သွားမှသာ (ဥပမာ ၂၀၀ မီလီစက္ကန့်ကြာမှ) လုပ်ငန်းစဉ်ကို လုပ်ခိုင်းသော နည်းလမ်းဖြစ်ပါသည်။

Debounce Code Concept

let timeout;
window.addEventListener('scroll', function() {
  clearTimeout(timeout);
  timeout = setTimeout(function() {
    console.log("Stopped scrolling. Execute code!");
  }, 200);
});

ဖြေရှင်းနည်း ၂: Throttling

Scroll ဆွဲနေစဉ်အတွင်း Event များကို တရစပ် အလုပ်မလုပ်စေဘဲ အချိန်တစ်ခု သတ်မှတ်၍ (ဥပမာ သုညဒသမ ၅ စက္ကန့်လျှင် တစ်ကြိမ်သာ) အလုပ်လုပ်စေသော နည်းလမ်းဖြစ်ပါသည်။

Throttle အလုပ်လုပ်ပုံ

Debounce သည် ရပ်သွားမှ အလုပ်လုပ်သော်လည်း၊ Throttle သည် အချိန်ကိုက်၍ (Timer ဖြင့်) အလုပ်လုပ်ပေးသောကြောင့် Infinite Scroll ကဲ့သို့သော နေရာများတွင် ပိုသင့်တော်ပါသည်။

requestAnimationFrame အသုံးပြုခြင်း

Scroll Animation များအတွက် အကောင်းဆုံးနှင့် အချောမွေ့ဆုံး ဖြေရှင်းနည်းမှာ window.requestAnimationFrame() ကို အသုံးပြုခြင်းဖြစ်ပါသည်။ Browser ၏ Screen Refresh Rate နှင့် အညီ Animation ကို လုပ်ဆောင်ပေးပါသည်။

Passive Event Listeners

Scroll ဆွဲရာတွင် ပိုမိုမြန်ဆန်ချောမွေ့စေရန် `{ passive: true }` ကို ထည့်သုံးသင့်ပါသည်။

window.addEventListener('scroll', myFunc, { passive: true });

(Browser ကို event.preventDefault() မသုံးပါဘူးလို့ ကြိုပြောထားခြင်းဖြစ်သည်)

အနှစ်ချုပ် (Summary)

  • window.scrollY ဖြင့် အပေါ်မှစ၍ scroll ပမာဏကို သိနိုင်သည်။
  • Sticky Nav, Infinite Scroll, Animations များအတွက် အဓိကသုံးသည်။
  • Performance အတွက် Debounce (သို့) Throttle ကို အမြဲစဉ်းစားပါ။
  • EventListener ထဲတွင် {passive: true} ကို ထည့်သုံးပါ။
